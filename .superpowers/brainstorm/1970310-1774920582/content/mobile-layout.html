手机端屏幕布局

终端为主的情况下，屏幕空间怎么分配？

A: 全屏终端 + 底部工具栏
Session 1 ● Connected
root@server:~/project$ claude
╭─────────────────────────╮
│ Claude Code │
╰─────────────────────────╯
> 帮我修改 login 组件...
I'll update the login...
root@server:~$ ▊
ESC TAB Ctrl ↑ ↓ ← →
输入命令...
🎤
终端占满全屏，底部固定工具栏
文件浏览通过手势或按钮切换
B: 终端 + 可滑出侧栏
☰ 文件 Session 1 ●
root@server:~/project$ claude
╭─────────────────────────╮
│ Claude Code │
╰─────────────────────────╯
> 帮我修改 login 组件...
I'll update the login...
root@server:~$ ▊
ESC TAB Ctrl ↑ ↓ ← →
输入命令...
🎤
左侧边缘右滑呼出文件侧栏
顶部汉堡菜单也可切换

💡 两者都以终端全屏为主。区别在于文件浏览器的呼出方式：A 通过顶部按钮切换视图，B 通过侧滑手势呼出覆盖层。
